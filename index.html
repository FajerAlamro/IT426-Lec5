Core Rules — What You Must Know
Greedy
f(n) = h(n)
  • Evaluates nodes using heuristic h(n) only
  • Always expands the node closest to the goal
  • Priority queue ordered by h(n) — min-heap
  • Not optimal — ignores path cost g(n)
  • Complete in finite spaces (graph search)
  • May loop in tree search (no explored set)
  • Time & space complexity: O(bm)
A* Search
f(n) = g(n)+h(n)
  • Combines actual cost g(n) with estimate h(n)
  • Expands the node with the smallest f(n)
  • Optimal if h is admissible: h(n) ≤ h*(n)
  • Complete — always finds a solution
  • Keeps all nodes in memory
  • Updates frontier when a cheaper path is found
  • SLD is always an admissible heuristic
